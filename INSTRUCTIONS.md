# GitHub 个人主页装扮指南

你好！这是一个为你准备的 GitHub 个人主页（Profile）装扮模板。
只要按照以下步骤操作，你就能拥有一个酷炫的个人主页了。

## 第一步：创建特殊仓库

1. 登录你的 GitHub 账号。
2. 点击右上角的 `+` 号，选择 **New repository**。
3. **关键点**：在 **Repository name** 中输入你的 **GitHub 用户名**。
   - 例如：如果你的用户名是 `zhangsan`，仓库名也必须是 `zhangsan`。
   - 你会看到一只猫出现并提示你：*You found a secret! ...*
4. 确保选择 **Public**（公开）。
5. 勾选 **Add a README file**。
6. 点击 **Create repository**。

## 第二步：应用模板

### 方式一：直接复制（推荐新手）

1. 打开你刚才创建的仓库页面。
2. 点击 `README.md` 文件旁边的 **铅笔图标** (Edit file)。
3. 将本文件夹中的 `README.md` 的内容 **全部复制** 并覆盖 GitHub 上的内容。
4. **修改内容**：
   - 将 `Your Name` 替换为你的昵称。
   - 将所有的 `your-username` 替换为你的 GitHub 真实用户名（注意：链接里有很多处，请仔细查找）。
   - 修改 `Tech Stack` 部分的图标（如果你不喜欢某些技术，可以删除对应的 `<img>` 标签）。
   - 修改底部的社交链接。
5. 点击页面底部的 **Commit changes** 保存。

### 方式二：使用 Git 上传（进阶）

如果你熟悉 Git，可以将本文件夹中的所有文件（包括 `.github` 目录）复制到你的本地仓库中，然后 push 到远程。

## 第三步：激活“贪吃蛇”动画（可选）

我在模板中为你准备了一个有趣的“贪吃蛇”动画，它会根据你的代码提交记录（绿格子）来生成动画。

1. **上传 Action 文件**：
   - 确保你已经把 `.github/workflows/snake.yml` 文件上传到了你的仓库中（如果你是用“方式一”，你需要手动在仓库里点击 **Add file -> Create new file**，输入路径 `.github/workflows/snake.yml`，然后把内容复制进去）。

2. **等待运行**：
   - 这个脚本设定为每 24 小时运行一次，或者在你推送到主分支时运行。
   - 你也可以在仓库的 **Actions** 标签页中，找到 "Generate Snake"，然后点击 **Run workflow** 手动触发一次。

3. **显示在主页**：
   - 等待 Action 运行成功后（变成绿色对勾）。
   - 回到你的 `README.md` 编辑页面。
   - 找到被注释掉的“贪吃蛇”部分（搜索 `snake`），去掉 `<!--` 和 `-->` 注释符号。
   - **注意**：确保图片链接中的 `your-username` 已经改成了你的用户名。

## 常见问题

- **图片加载不出来？**
  - 检查链接中的 `your-username` 是否替换正确。
  - GitHub 的图片缓存可能需要几分钟刷新。
